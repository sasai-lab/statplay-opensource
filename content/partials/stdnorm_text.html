// よくある誤解

❌「z=1.96 はキリのいい数字だから採用された」

1.96 は「両側確率がちょうど 5% になる z の値」として計算で出てきた数字。キリが良いわけでもなく、約束事でもない。P(−1.96 ≤ Z ≤ 1.96) = 0.9500 は近似。試験では 1.96 で十分。

❌「標準化したらデータの情報が消える」

z = (x−μ)/σ は可逆。x = μ + zσ で元に戻せる。標準化はスケールを揃えるだけで、情報は一切失われない。

❌「標準正規分布は理論だけのもの」

どんな正規分布も z = (x−μ)/σ で標準正規に変換できるから、この1つの分布を知っていれば全部の確率が計算できる。だからこそ「すべてのはじまり」。

// Common misconceptions

❌ "z = 1.96 is a round number chosen by convention"

1.96 is the z-value where the two-sided probability equals exactly 5%. It wasn't chosen for convenience — it came straight out of the math. P(−1.96 ≤ Z ≤ 1.96) = 0.9500 is approximate. For exams, 1.96 is precise enough.

❌ "Standardizing destroys information"

z = (x−μ)/σ is reversible: x = μ + zσ recovers the original value. Standardization rescales; it doesn't discard anything.

❌ "The standard normal is purely theoretical"

Any normal distribution transforms to the standard normal via z = (x−μ)/σ. Know this one distribution and you can compute probabilities for all of them. That's why it's "where everything begins."

// 試験での出方

  • 表の読み方：問題の表が「上側確率」か「累積確率」か確認。間違えると答えが全然違う
  • 引き算パターン：P(0.5 ≤ Z ≤ 1.5) = Φ(1.5) − Φ(0.5)
  • 標準化してから表を引く：「N(170, 6²) で 180 以上の確率」→ z = (180−170)/6 ≈ 1.67 → 表を引く
  • 逆引き：「上側5%点は？」→ z = 1.645

// How it shows up in exams

  • Reading the table: check whether the table gives "upper-tail" or "cumulative" probabilities — mixing them up changes the answer completely
  • Subtraction pattern: P(0.5 ≤ Z ≤ 1.5) = Φ(1.5) − Φ(0.5)
  • Standardize then look up: "P(X ≥ 180) where X ~ N(170, 6²)" → z = (180−170)/6 ≈ 1.67 → look up the table
  • Reverse lookup: "What is the upper 5% point?" → z = 1.645
