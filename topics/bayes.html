ベイズ定理 — 陽性的中率を直感で

有病率・感度・特異度を動かし、陽性と診断された人が実際に病気である確率（陽性的中率）がどう変わるかを可視化。偽陽性の多さに驚くベイズの直感を掴む。

P.05 / BAYES THEOREM

ベイズの定理 — 事後確率の逆転劇

分布の道具箱が揃った。最後に条件付き確率を反転させる——これがベイズの定理。検査で陽性が出た→病気の確率は？ ここで直感はよく裏切られる。

「感度99%・特異度95%の検査で陽性」＝ 99%病気？
…答え：わずか 16.7%。自分も最初「え、99%じゃないの？」と思っていました。医師でも半分以上が外す有名クイズなので、直感とズレても自分だけじゃない、というのが救いです。
ポイントは"もともと病気の人がめっちゃ少ない"という事実を忘れてしまうこと。 下の"1000人の町"を見ながら、3つのつまみを動かして自分の目で確かめよう。

実験ガイド — 順番に試してみよう
  1. Step 1: 初期設定（有病率1%, 感度99%, 特異度95%）で陽性的中率を確認 → たった16.7%。直感とのズレを味わう。
  2. Step 2: 有病率を 50% に上げる → 陽性的中率が一気に跳ね上がる。事前確率の効き方がここまで強いとは、自分も動かしてみて初めて実感しました。
  3. Step 3: 有病率を1%に戻し、特異度を 99.9% にする → 偽陽性が激減し、陽性的中率が改善。
  4. Step 4: 1000人の町の図を見ながら、TP（真陽性）と FP（偽陽性）の人数比を確認しよう。

// ここで使われる公式

各パーツの役割（検査の例で読む）
・P(A|B)：「陽性が出たとき、本当に病気である確率」← これが知りたい
・P(B|A)（尤度）：「病気の人が検査で陽性になる確率」= 感度
・P(A)（事前確率）：検査する前の「そもそも病気である確率」= 有病率
・P(B)（周辺確率）：「検査で陽性が出る確率」全体。病気でも健康でも含む

なぜ直感が外れるのか
・P(A) が小さい（有病率が低い）とき、分子 P(B|A)·P(A) はどんなに感度が高くても小さい
・一方 P(B) には健康な人の偽陽性も含まれるので意外と大きい
・結果：小さい数 ÷ そこそこ大きい数 = 「陽性でも病気じゃないことが多い」

// よくある誤解

❌「陽性 = 病気」

有病率1%, 感度95%, 特異度90% だと、陽性的中率はたった 8.8%。陽性100人中、実際に病気なのは9人だけ。残りの91人は健康なのに陽性（偽陽性）。上のシミュレーションで有病率を下げていくと、この比率が目で見てじわじわ腑に落ちてきます。

❌「感度と特異度が高ければ安心」

有病率が低すぎる集団に使うと、感度・特異度が両方99%でも陽性的中率は50%程度。「誰に対して検査するか」（事前確率の選び方）が結果を大きく左右する。

❌「ベイズの定理は1回使って終わり」

1回目の事後確率を2回目の事前確率として更新できる。2回連続で陽性なら、事後確率はぐっと上がる。これが「ベイズ更新」の考え方。

// 試験での出方

  • 定番計算：「有病率 p%, 感度 a%, 特異度 b%。検査陽性のとき病気の確率は？」
  • 全確率の公式：P(B) = P(B|A)P(A) + P(B|Ā)P(Ā) を分母に使う
  • 自然頻度で解く：1000人の表を書いて直接数えるのが確実。公式を暗記するより速い
