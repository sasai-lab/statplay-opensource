## t分布を描きたかっただけなのに

事の発端は、t分布のグラフを描きたかったことでした。

StatPlay には正規分布のセクションがあって、そこでは `normPDF`（正規分布の確率密度関数）を使ってグラフを描いています。正規分布の式は比較的シンプルで、JavaScriptの標準的な数学関数だけで書けます。

```javascript
// 正規分布の確率密度関数 — Math.exp, Math.sqrt, Math.PI だけで書ける
export function normPDF(x, mu = 0, sd = 1) {
  return Math.exp(-0.5 * ((x - mu) / sd) ** 2) / (sd * Math.sqrt(TAU));
}
```

ところが、t分布を描こうとしたときに事件が起きました。AIに「t分布のグラフも描いて」と指示したら、こういうコードが出てきたんです。動きました。グラフも出ました。でも後から「これ何やってるんだ」とコードを読み返したら、まったく意味がわからない。

式にするとこうです。

$$
f(x; \nu) = \frac{\Gamma\left(\frac{\nu+1}{2}\right)}{\sqrt{\nu\pi}\,\Gamma\left(\frac{\nu}{2}\right)} \left(1 + \frac{x^2}{\nu}\right)^{-\frac{\nu+1}{2}}
$$

$\Gamma$ って何だ。動いてるけど、自分が何を動かしているのかわからない。これはまずい。

## ガンマ関数とは何か

AIに聞きました。「この式に出てくるΓって何ですか」と。

教えてもらった内容を自分なりに噛み砕くと、こういうことでした。

階乗（$n!$）は知っています。$5! = 5 \times 4 \times 3 \times 2 \times 1 = 120$。ただ、これは自然数でしか使えません。$3.5!$ とか $0.5!$ は定義できない。

ガンマ関数は、この階乗を実数（と複素数）に拡張したものです。自然数 $n$ に対して $\Gamma(n) = (n-1)!$ が成り立ちます。つまり $\Gamma(5) = 4! = 24$ です。

なんで $\Gamma(5) = 5!$ じゃなくて $(5-1)!$ なのかというと、歴史的な経緯でそうなっているらしいです。最初に知ったときは「なんでひとつズレてるんだ」と思いましたが、まあそういうものだと受け入れました。

で、t分布の式にガンマ関数が出てくるのは、自由度が整数じゃないケースにも対応するためです。$\Gamma(\nu/2)$ の $\nu$ が奇数だと、$\Gamma(2.5)$ とか $\Gamma(3.5)$ みたいな値が必要になる。階乗では計算できない。だからガンマ関数が要る。

ここまではわかりました。問題は、**JavaScriptにはガンマ関数がない** ということでした。

Python には `math.lgamma()` がある。R には `lgamma()` がある。でも JavaScript の `Math` オブジェクトにはない。自分で実装するしかない。

---

## Lanczos近似という方法

ガンマ関数を実装する方法はいくつかあるらしいですが、AIと相談した結果、Lanczos近似を使うことにしました。

正直に言うと、Lanczos近似の数学的な導出は理解しきれていません。ただ、「なぜこの方法を選んだか」と「何をやっているか」の概要は把握しています。

Lanczos近似は、ガンマ関数を近似計算するためのアルゴリズムで、広い範囲の入力に対して高い精度が出ます。Wikipedia にも載っていて、Python の `math.lgamma` や多くの数値計算ライブラリでも同じ方法が使われているとのことでした。

実装では、ガンマ関数そのものではなく **対数ガンマ関数**（$\ln\Gamma(z)$）を計算しています。なぜかというと、ガンマ関数は値がすぐに巨大になるからです。$\Gamma(100)$ はもう `Infinity` になってしまう。対数を取っておけば桁あふれを防げます。最終的に値が必要なときに `Math.exp(lgamma(z))` で戻せばいい。

これが実際のコードです。

```javascript
// 対数ガンマ関数 — Lanczos近似
// 定数 g=7 と係数配列 c[] は近似精度を決めるパラメータ
export function lgamma(z) {
  const g = 7;
  const c = [
    0.99999999999980993,    // c[0]: 級数の初項
    676.5203681218851,      // c[1]〜c[8]: 近似の精度を上げる係数
    -1259.1392167224028,
    771.32342877765313,
    -176.61502916214059,
    12.507343278686905,
    -0.13857109526572012,
    9.9843695780195716e-6,
    1.5056327351493116e-7
  ];

  // z < 0.5 のとき：オイラーの反射公式で正の領域に変換する
  // Γ(z)Γ(1-z) = π / sin(πz) を使う
  if (z < 0.5) return Math.log(Math.PI / Math.sin(Math.PI * z)) - lgamma(1 - z);

  z -= 1;

  // Lanczos級数の計算
  let x = c[0];
  for (let i = 1; i < g + 2; i++) x += c[i] / (z + i);

  const t = z + g + 0.5;

  // 最終的な対数ガンマ値を返す
  return 0.5 * Math.log(TAU) + (z + 0.5) * Math.log(t) - t + Math.log(x);
}

// ガンマ関数本体 — 対数ガンマから復元する
export function gamma(z) {
  return Math.exp(lgamma(z));
}
```

正直、係数配列 `c` の数値がなぜこの値なのかは完全には理解できていません。Lanczos近似の理論から導出される値で、$g=7$ のときにこの9個の係数を使うと、倍精度浮動小数点の範囲で十分な精度が出る、ということは確認しました。

ひとつ面白かったのは、`z < 0.5` のときの処理です。これはオイラーの反射公式というもので、$\Gamma(z)\Gamma(1-z) = \pi / \sin(\pi z)$ という関係を使って、負の領域を正の領域の計算に変換しています。再帰呼び出しになっていて、`lgamma(1 - z)` を呼ぶと今度は引数が 0.5 以上になるので、本体の計算に入る。これは「なるほど」と思いました。

……と書いていて気づいたんですが、Lanczos近似だのオイラーの反射公式だの、統計検定2級の試験範囲なんてとっくの昔に通り過ぎています。そもそもオイラーって誰ですか。統計の勉強をしていたはずなのに、いつの間にか18世紀の数学者の公式を実装している。局地的にではありますが、明らかにヤバい方向に進んでいます。

## t分布・カイ二乗分布・F分布が書けるようになった

`lgamma` が実装できたことで、t分布・カイ二乗分布・F分布の確率密度関数がすべて書けるようになりました。

```javascript
// t分布 — 自由度dfのt分布の確率密度関数
export function tPDF(x, df) {
  // lgammaのおかげで Γ((df+1)/2) / Γ(df/2) が計算できる
  return Math.exp(lgamma((df + 1) / 2) - lgamma(df / 2))
    / Math.sqrt(df * Math.PI)
    * Math.pow(1 + x * x / df, -(df + 1) / 2);
}

// カイ二乗分布 — 自由度dfのχ²分布の確率密度関数
export function chi2PDF(x, df) {
  if (x <= 0) return 0;
  return Math.exp(
    (df / 2 - 1) * Math.log(x) - x / 2 - (df / 2) * Math.log(2) - lgamma(df / 2)
  );
}

// F分布 — 自由度(d1, d2)のF分布の確率密度関数
export function fPDF(x, d1, d2) {
  if (x <= 0) return 0;
  // ベータ関数 B(d1/2, d2/2) = Γ(d1/2)Γ(d2/2) / Γ((d1+d2)/2)
  const num = d1 / 2 * Math.log(d1 * x) + d2 / 2 * Math.log(d2)
    - (d1 + d2) / 2 * Math.log(d1 * x + d2);
  const beta = lgamma(d1 / 2) + lgamma(d2 / 2) - lgamma((d1 + d2) / 2);
  return Math.exp(num - beta) / x;
}
```

どの関数も、計算の途中で `lgamma` を使って対数の世界で足し引きして、最後に `Math.exp` で戻すという構造になっています。こうすることで、途中の値が巨大になっても桁あふれしない。

これはAIに教えてもらって「なるほど」と思ったテクニックでした。ガンマ関数の値そのものは掛け算で使うことが多いんですが、対数を取れば掛け算が足し算になる。だから `lgamma` を先に作っておけば、あとは足し引きだけで済む。というか、こんなの自分で思いつくのは不可能です。

## この計算、本当に合ってる？

コードが動くことと、コードが正しいことは別の話です。ここが一番神経を使ったところでした。

### ガンマ関数の基本恒等式

まず、ガンマ関数が満たすべき数学的な性質を検証しました。

```javascript
// T128: Γ(n) = (n-1)! が成り立つか
// Γ(1) = 0! = 1
// Γ(5) = 4! = 24
// Γ(0.5) = √π（これは有名な値）
ok('T128 gamma: Γ(1)=1, Γ(5)=24, Γ(0.5)≈√π',
   Math.abs(gamma(1) - 1) < 1e-10 &&
   Math.abs(gamma(5) - 24) < 1e-8 &&
   Math.abs(gamma(0.5) - Math.sqrt(Math.PI)) < 1e-8);
```

$\Gamma(1) = 1$、$\Gamma(5) = 24$、$\Gamma(0.5) = \sqrt{\pi}$。最初の二つは階乗との関係で明らかですが、$\Gamma(0.5) = \sqrt{\pi}$ は非整数の値なので、Lanczos近似が非整数でもちゃんと動くことの検証になります。誤差は $10^{-8}$ 未満でした。

### t分布の性質

t分布については、数学的にわかっている性質をテストしました。

```javascript
// T124: 自由度が大きくなると正規分布に近づく
// tPDF(1.5, df=200) ≈ normPDF(1.5)
ok('T124 tPDF(1.5, df=200) ≈ normPDF(1.5)',
   Math.abs(tPDF(1.5, 200) - normPDF(1.5)) < 1e-3);

// T125: t分布はx=0を中心に左右対称
// tPDF(-x, df) = tPDF(x, df) が成り立つか
ok('T125 tPDF symmetry',
   Math.abs(tPDF(-2, 5) - tPDF(2, 5)) < 1e-15 &&
   tPDF(0, 5) > tPDF(1, 5));
```

「自由度が大きくなるとt分布は正規分布に近づく」というのは教科書に書いてある性質です。df=200 で正規分布との差が $10^{-3}$ 未満。対称性は $10^{-15}$ 未満。

### カイ二乗分布・F分布

```javascript
// T126: χ²分布のピークは (df-2) 付近にある（df≥2のとき）
// df=10 → mode=8 → chi2PDF(8, 10) > chi2PDF(6, 10)
ok('T126 chi2PDF peak near mode', chi2PDF(8, 10) > chi2PDF(6, 10));

// T127: F分布は正の入力で正の有限値、x≤0で0を返す
ok('T127 fPDF positive and finite',
   fPDF(2, 5, 10) > 0 && isFinite(fPDF(2, 5, 10)) &&
   fPDF(0, 5, 10) === 0 && fPDF(-1, 5, 10) === 0);
```

これらのテストは GitHub Actions で毎プッシュ時に走ります。何かを変更して精度が落ちたら、すぐに気づけます。

## 理解したから検証できる

t分布を描きたかっただけなのに、ガンマ関数の実装が必要になり、階乗の拡張とは何かを学び、対数の世界で計算する理由を理解し、オイラーの反射公式の存在を知りました。

AIがコードを書いてくれたのは確かです。でも、そのコードが何をやっているかを理解し、なぜこの係数なのか、なぜ対数を使うのかを自分で納得するまでには、けっこうな時間がかかりました。対数の性質が怪しくなるたびにAIに高校数学のレベルから教えてもらっていました。

理解したからこそ前のセクションのような検証ができる。検証できるからこそ「この計算は正しい」と言える。
